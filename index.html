Flatland Project

The Flatland HTML website further demonstrated the event driven model using JavaScript. The purpose of this website is that if the user clicks on the square while hovering it will update the text, generating a random buzz word. Furthermore, when the user hovers over the square it will change colour. Theses two main functions were driven by JavaScript, and styled using CSS.

To-Do List

The To Do List website demonstrates local storage which works in conjunction with user input from the buttons. The inputs interact with the JavaScript to update the webpage and store the local data on the server. We used the DOM application the plan the basics, which then resulted in a simple list of use cases that we may want to include in our To Do List Website (Add checkboxes, database, simple list item, add a form, add style). The user can add a task to the list, then click on the checkbox to mark it as completed. Using the buttons, the user can see "All" tasks, "Active" tasks, and "Completed" task.

Click Here to Vist the GitHub Repository

RSS Feeder

The RSS feeder was the website that included an XHR Request. We created an XHR object, write an event listener, and then open and send. The websites purpose is to feed the website an RSS link, and it will list all the RSS feeds in a page. Including the title, publication date, author, link and content.
